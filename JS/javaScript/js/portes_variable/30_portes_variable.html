La portée des fonctions et les conflits

La portée est un concept trés important en javascript lorsqu'on à faire des fonctions. Quand on crée une fonction , les variables et autres choses qui sont définies à l'intèrieur de la fonction ont leur propres portée. Ce qui signifie qu'elles sont enfermées dans leurs propres compartiment séparé et qu'elles ne peuvent pas être affectées par d'autres fonctions

Le plsu haut niveau en dehors de tooutes vos fonctions est appelé la portée globale. Les valeurs définies dans la "pg" sont accessible à partir de n'importe quel partie du code

Le JS est construit de cette façon pour une question de sécurité et d'organisation

Parfois, vous ne voulez pas que vos variables soient accessible depuis toutes les autres partie du code -script en externes mauvaise pratiques !!!!
