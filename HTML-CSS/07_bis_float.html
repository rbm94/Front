=>Notion de flux
Le flux d'un document pourrait se définir comme étant le comportement naturel d'affichage des éléments d'une page web. Autrement dit,les éléments se succèdent dans l'ordre où ils sont déclarés dans le code HTML. De façon simpliste,les éléments de séquençage tels que "h1","p",ou "ul", s'affichent logiquement par défaut les uns en dessous des autres tandis que les éléments délimintant pour la plupart des portions de trextes (mise en italique d'un ou plusieurs mot par ex.) s'affichent côte à côtes
. Maintenant que nous avons évoqué ce point,il faut comprendre que les balises HTML ont toutes par défaut des propriétés de rendu CSS particulières. En fait,il existe initialement deux grands groupes principaux de balises: les balises de rendu CSS"bloc"(block) et Les balises de rendu CSS "en-ligne("inline"). Ces valeurs de rendu visuel coïncident généralement par défaut avec le groupe d'appartenance HTML
. Des propriétés CSS d'un élément ( Display: block,inline, list-item, inline-block, table, table-cell,.... ) découlent des spécificités d'affichage, son positionement dans le flux.

Il est possible de sortir du flux naturel un ou plusieurs éléments pour modifier leur affichage avec du CSS

Nos deux groupes d'éléments peuvent êtres classés en deux familles: Les éléments de type block (h1, p, ul, ol, dl, table, blockquote,ect... Les élément de type inline ( a, img , strong, abbr, ect...)

=>Le type "block"

Les éléments de rendus CSS block se plus toujours l'un en dessous de l'autre par défaut par exemple, une suiye de paragraphe (Balise

.) par ailleurs, un élément "block" occupe automatiquement, par défaut, toute la largeur disponible dans son conteneur et peut-être fimensinné à l'aide des propriétés telles que "width", "height", "min-width", ou "min-height",....
De plus un éléments block peut contenir des éléments block et inline . A RETENIR !!!! Un élément de type "block" se différencie des éléments de type "inline"su différent points, dont ceux-ci:
  • Il occupe l'entièrté de la largeur de son conteneur
  • Il permet l'attribution de marges verticales
  • Il permet la modification de sa hauteur et de sa largeur.
Le premier point est primordial, nous utiliserons abondamment ce comportement par défaut bien pratique dans la création de ne mise en page. Notons également que tout élément peut être "reclassé" dans la familleopposée grace a la propriété display.

=>Le type "inline".
Les élément rendus "inline" se placent toujours l'un a côté de l'autre afin de rester dans le texte (balises stron, img, a, span, ect...). Par défaut , il n'est pas prévus qu'il puissent se positionner sur la page , ni de leur donner des dimmensions, leur taille va être déterminée par le texte ou lélément/image qu'ils contiennent. Référence:
La propriété CSS FLOAT permet de faire flotter des éléménts côte à côte, donc de modifier le comportement naturellement de type block en comportement inline . A l'origine le float est prévus pour entourer une image avec du texte(paragraphes).
